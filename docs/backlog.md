# Guiding Principle

Ideas are cheap.

Shipping is hard.

Everything goes into the backlog instead of being added immediately.

Features are prioritized based on user value, not technical novelty.

# Success Criteria Before Adding New Features

Before building anything new, ask:

1. Does it help users recreate the same photo?

2. Have at least five users requested it?

3. Does it improve the core experience rather than distract from it?

If the answer to any of these is "no," it stays in the backlog.
